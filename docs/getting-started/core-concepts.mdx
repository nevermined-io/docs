Nevermined helps you monetize **AI-enabled services** — whether that’s an agent, an MCP tool/server, or access to a protected resource (like a dataset or file). At the heart of the platform are two concepts:

- **Services & Resources**: the thing you’re charging for (an API, a tool, or gated access)
- **Payment Plans**: how customers pay and what they’re entitled to consume

## What is a Service or Resource?

In the Nevermined ecosystem, a *service* is anything you expose for paid access. That might be:

- **AI Agents**: autonomous or human-in-the-loop agents you call via HTTP.
- **MCP Tools / Servers**: monetizable tool endpoints accessed via the MCP.
- **Protected Assets**: gated resources like files, datasets, prompts, or static downloads.

Regardless of the form factor, a monetizable service/resource typically has:

- A **public interface** (HTTP endpoint, MCP tool, or download URL)
- A **pricing + access policy** (a payment plan)
- A **verification step** (validate entitlement before delivering)
- **Usage tracking** (meter each request or access)

## Standards and protocols (x402)

Nevermined supports modern payment standards used in AI and API commerce. In particular, **x402** standardizes how clients and servers negotiate payment-required requests, enabling smoother, interoperable monetization flows.

If you're integrating with x402-compatible clients/servers, see the [x402 integration](/docs/development-guide/nevermined-x402).

## What is a Payment Plan?

A Payment Plan is a set of rules and conditions that define how users pay to access your service/resource. Plans are defined by two main components: their **price** and the **consumption** model.

### Key Attributes of a Payment Plan

  <Card title="Payment Method" icon="credit-card">
    Choose whether customers pay in [**Fiat**](/docs/integrate/patterns/fiat-payments) (card delegation via [NVM Pay](/docs/products/nvm-pay/overview)) or [**Crypto**](/docs/integrate/patterns/stablecoin-payments) (stablecoins like USDC or EURC).
  </Card>
  <Card title="Pricing & Receivers" icon="sack-dollar">
    Define price, payout currency, and how revenue is distributed across one or more receivers.
  </Card>
  <Card title="Plan Types" icon="calendar-days">
    - **Plan**: set a price for a bundle of API calls, with a fixed or dynamic cost per request
    - **Pay-as-you-go**: charge the user's payment method on each request
    - **Time-based**: access limited by duration (e.g., 30 days)
    - **Trial**: free limited access for testing
  </Card>
  <Card title="Settlement" icon="coins">
    Choose how payments settle:

    - **Fiat**: via card delegation ([NVM Pay](/docs/products/nvm-pay/overview))
    - **Crypto**: stablecoins (USDC, EURC) on-chain
  </Card>
</CardGroup>

<AccordionGroup>
  <Accordion title="Plans (prepaid bundles)">
    The most common model. Set a price for a bundle of API calls and define the cost per request:

    - **Fixed pricing**: every request costs the same (e.g., $10 for 100 requests at $0.10 each)
    - **Dynamic pricing**: set a range and charge different amounts per request based on complexity or resource usage
  </Accordion>
  <Accordion title="Pay-as-you-go">
    Charge the user's payment method on each request, no upfront purchase:

    - **Per-request settlement**: charge as each request happens
    - **Card delegation**: users enroll a card via [NVM Pay](/docs/products/nvm-pay/overview) and agents charge it directly
    - **Stablecoins**: settle in USDC or EURC on-chain
  </Accordion>
  <Accordion title="Time-based plans">
    Ideal for subscription models:

    - **Duration**: set expiration time (days, months, years)
    - **Unlimited usage**: within the time period
    - **Auto-expiry**: plan expires after duration
  </Accordion>
  <Accordion title="Hybrid plans">
    Combine time limits with usage limits:

    - **Usage with expiry**: e.g., 1,000 requests valid for 30 days
    - **Rate limiting**: control usage within time periods
  </Accordion>
</AccordionGroup>

### Pricing Models

Beyond plan types, you can choose how pricing maps to the value you deliver:

<CardGroup cols={2}>
  <Card title="Cost + Margin" icon="calculator">
    Bill per API call or per session with full cost observability. Set your margin on top of compute costs. Best for predictable, high-volume workloads.
  </Card>
  <Card title="Outcome-Based" icon="bullseye">
    Charge on success: meetings booked, leads qualified, issues resolved. Only bill when your agent delivers a defined outcome or milestone.
  </Card>
</CardGroup>

### Example Payment Plans

- **Customer Support MCP Tool**: $49/month for unlimited usage, or $10 for 1,000 API calls.
- **Dataset Access**: $99 for 30-day access to a protected resource.
- **Agent API**: $10 for 100 requests at $0.10 per call, or pay-as-you-go at $0.15 per call.
- **Sales Agent**: $5 per qualified lead (outcome-based).

## How It All Works Together

<Steps>
  <Step title="Register the service/resource">
    Define your service interface (API, MCP tool, or protected asset) and metadata.
  </Step>
  <Step title="Create plans">
    Create one or more payment plans: set pricing, cost per request, and duration.
  </Step>
  <Step title="Customers purchase access">
    Users buy a plan (fiat or crypto) or enroll a card for pay-as-you-go via [NVM Pay](/docs/products/nvm-pay/overview).
  </Step>
  <Step title="Validate before delivery">
    Validate entitlement before serving requests or granting access.
  </Step>
  <Step title="Track and settle">
    Automatically track usage per request and settle payments.
  </Step>
</Steps>

By combining services/resources with flexible payment plans, Nevermined supports monetization across agents, MCP tools, and protected assets.

## Payment Methods

Nevermined supports three payment methods, each optimized for different audiences:

<CardGroup cols={3}>
  <Card title="Fiat Payments" icon="credit-card" href="/docs/integrate/patterns/fiat-payments">
    Pay with real cards via [NVM Pay](/docs/products/nvm-pay/overview) card delegation. No crypto wallet needed, the fastest path for mainstream and enterprise adoption.
  </Card>

  <Card title="Stablecoin Payments" icon="coins" href="/docs/integrate/patterns/stablecoin-payments">
    Accept USDC, EURC, USDT, or any ERC-20 token on-chain. Lowest fees and full transparency for crypto-native users and agent-to-agent commerce.
  </Card>

  <Card title="Card Delegation (NVM Pay)" icon="id-card" href="/docs/products/nvm-pay/overview">
    Users enroll a Visa or Stripe card and create spending mandates. Agents charge directly against these mandates via x402, with per-transaction ceilings, usage limits, and expiration controls.
  </Card>
</CardGroup>

### Merchant Onboarding

Fiat settlement is merchant-centric. To receive payouts, create a [Stripe Connect](https://stripe.com/connect) account and link it to Nevermined. This is the only setup required on the builder side. Stablecoin payouts settle directly to your wallet. Additional PSPs are on the roadmap.
